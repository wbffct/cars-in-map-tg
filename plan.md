## Текущий статус

Собран рабочий прототип Telegram Mini App:

- карта с машинами загружается из публичного `cars.json`
- список машин автоматически обновляется каждые 2 минуты
- пины отображаются на карте
- по нажатию на пин открывается карточка поверх карты
- в карточке показывается номер автомобиля
- кнопка переводит пользователя в приложение по deeplink
- добавлено базовое логирование пользовательских действий

## Что уже реализовано

### Frontend

- `React + TypeScript + Vite`
- `MapLibre GL` для карты
- интеграция с Telegram WebApp bridge
- локальный запуск через `npm run dev`

### UX

- на экране только карта
- карточка открывается как bottom sheet
- карточка закрывается по крестику или тапу вне нее
- цвет пинов: `#2AE1B9`

### Данные

Текущий источник данных: `public/cars.json` или внешний URL через `VITE_CARS_URL`

Используемые поля:

- `car_id`
- `latitude`
- `longitude`
- `hex`
- `license_plate_number`

### Deeplink

Формат deeplink:

```text
https://app.delitime.kz/auto/{LICENSE_PLATE_NUMBER}
```

Номер автомобиля передается в верхнем регистре.

Пример:

```text
https://app.delitime.kz/auto/P213702
```

### Логирование

Сейчас логируются события:

- `mini_app_open`
- `map_ready`
- `pin_click`
- `card_close`
- `deeplink_click`

На текущем этапе события пишутся:

- в `console.info`
- в `window.__miniAppLogs__`

### 1. Подключить нормальную аналитику

Варианты:

- отправка событий в backend endpoint
- Amplitude
- любой внутренний аналитический сервис

### 2. Перейти с локального JSON на backend/API

Нужен endpoint, который будет возвращать список машин в формате для Mini App.

Минимальный ответ:

```json
[
  {
    "car_id": 40663,
    "latitude": 43.20636,
    "longitude": 76.86542,
    "hex": "8820e60b5bfffff",
    "license_plate_number": "p213702"
  }
]
```

### 3. Расширить карточку машины

Когда появятся данные, можно добавить:

- модель
- заряд / топливо
- бонус
- адрес
- фото

### 4. Добавить fallback-сценарий

Если deeplink не открывает приложение:

- или открыть web landing
